# 한국 화물운송 시장: 플랫폼 및 배차 시스템 현황 보고서

## 1. 개요

한국 화물운송 시장은 **화주(Shipper) → 주선업자(Broker) → 운송업자/차주(Operator)** 의 3단계 구조를 기본으로 하며, 디지털 플랫폼의 급속한 확산으로 전통적인 배차 시스템이 빠르게 혁신되고 있는 상황입니다. 국내 육로 화물운송 물량의 약 80% 이상이 약 7,000여 개의 주선업체를 통해 처리되고 있습니다.

## 2. 주요 화물운송 플랫폼 현황

### 2.1 로지스팟 (LOGISPOT)

#### 기본 정보
- **위치**: 국내 디지털 통합물류 시장 1위 기업
- **설립 목표**: 화물운송 산업의 디지털화 선도
- **비즈니스 모델**: B2B 운송 플랫폼 (미들마일 물류)

#### 주요 특징
- **계약 화주**: 약 700여 개 기업
- **배차 구조**:
  - 자사 네트워크: 42%
  - 제휴 협력사: 58% (5~6개 운송사)
  - 부족 물량: 화물정보망에 공고

- **서비스 특성**:
  - 화물운송을 디지털 환경에서 수행하여 데이터 축적
  - 운송 효율성과 투명성 극대화
  - "바로배차" 서비스: 기업이 즉시 배차 가능

#### API 지원 현황
**로지스팟 오픈 API (2019년 공개)**

로지스팟은 업계 최초로 기업의 내부 시스템과 연동 가능한 오픈 API를 공개했습니다.

**API의 정의와 기능**:
- 기업의 전사적 자원관리 시스템(ERP)과 운송 플랫폼 연동
- 운송 정보 확인 및 관리 자동화
- 실시간 데이터 연계

**연동 가능 시스템**:
- ERP (전사적 자원관리 시스템)
- WMS (창고관리 시스템)
- OMS (주문관리 시스템)
- 이커머스 플랫폼
- 풀필먼트 시스템
- 3PL 파트너사
- 포워딩 업체

**대상 산업**:
- 식품, 제조, 유통, 이커머스 등 다양한 분야

### 2.2 티맵 화물 (TMAP Freight / YLP 화물)

#### 기본 정보
- **운영사**: T맵모빌리티 (카카오 계열사)
- **서비스명**: YLP 화물 (TMAP Freight)
- **출시**: 2023년 정식 출시
- **비즈니스 모델**: 화주 중심의 미들마일 배차 플랫폼

#### 주요 특징
- **배차 성공률**: 94% (CBT 기간 중 기존 90% → 94%로 상승)
- **책임 배차**: 100% 완전 배차 보장
- **배차 방식**:
  - 외부망과 연결된 자동 배차 시스템
  - 전국 8개 지사 협력체계
  - 실시간 배차 최적화

#### 핵심 경쟁력: "최적 운임 조회" 서비스
- **빅데이터 활용**: 110만 개 이상의 운송 빅데이터
- **실시간 분석 요소**:
  - 기상 정보
  - 전국 화물차 수요·공급 현황
  - 도로 교통 상황
- **최적 운임료 자동 산출**

#### 서비스 통합 흐름
```
운임 조회 → 접수 → 배차 → 정산 (원스톱 서비스)
```

#### 접근성
- PC, 모바일 모두 지원
- 맞춤 견적 제공
- 고정차 배차 옵션
- Open API 제공

### 2.3 기타 주요 플랫폼

#### 배차킹 (Dispatch King)
- **특징**: 초간편 스마트 화물운송 플랫폼
- **타겟**: 중소 운송사 및 차주

#### 전통적 화물정보망 (Freight Information Network)
- **역할**: 차주가 화주/운송사/주선사 구분 없이 오더를 받을 수 있는 기본 인프라
- **이용률**: 2017년 기준 운송주선사 차량확보 방법 중 화물정보망 활용 비중 44.3% (전년 대비 18.3% 증가)
- **형태**: 모바일앱, 웹 기반 플랫폼

#### 핵심 기능
1. **주문 통합**: 수백 개의 쇼핑몰과 중개업체 자동 연동

3. **전 프로세스 자동화**:
   - 주문 입력 → 창고 관리 → 택배 배송 → 라스트마일 배송
4. **실시간 추적**: 배송 상태 실시간 업데이트

#### 배차 및 자동화 기술
- **RFID 자동화**: 물류센터의 상차/입고 검수 자동화
- **접안슈트 자동배정**: 간선차량 출입 시 프로그램에 의해 자동으로 배정, 기사에게 문자 통보
- **통합 배송**: 주문부터 출고까지 전 과정 관리

#### API 연동
- **택배 연동 방식**: 계약 고객이 대리점에 "상품추적전달 등록" 요청
- **표준 API 활용**: 타 플랫폼과의 빠른 연계 가능
- **통합 송장 관리**: 송장번호 및 배송상태 자동 연동

## 4. 운송 프로세스 및 배차 시스템

### 4.1 전체 운송 체인 (Supply Chain)

```
화주 (Shipper)
  ↓
  주선업자 (Freight Broker)
    ├─ 자체 운송사 (Carrier)
    ├─ 협력 운송사 네트워크
    └─ 화물정보망 공고
  ↓
운송사 / 차주 (Operator / Driver)
  ↓
수취인 (Consignee)
```

### 4.2 배차 프로세스 상세 단계

#### 1단계: 주문 접수
- **시작점**: 화주로부터 운송 의뢰 발생
- **입력 사항**:
  - 화주 정보
  - 품목 정보
  - 출발지/도착지
  - 배송 요청사항

#### 2단계: 기초자료 활용
- **거래처 관리**: 화주 정보 데이터베이스
- **차량 관리**: 보유 차량 정보 및 운행 현황

#### 3단계: 배차 수행
- **단계 1**: 자사 네트워크 우선 배차
- **단계 2**: 제휴 협력사 배차
- **단계 3**: 화물정보망 공고

#### 4단계: 화물 공개
- **플랫폼**: 화물정보서비스 (FIS)
- **대상**: 독립 차주 및 소규모 운송업자
- **방식**: 오픈 입찰 형태

#### 5단계: 운송 실행 및 모니터링
- **배차 확정**: 차주 수락 또는 배차 강제
- **실시간 추적**: 위치 추적 및 배송 상태 업데이트
- **증거 자료**: 픽업/배송 확인

### 4.3 TMS (Transportation Management System)의 역할

#### 정의
운송 관련 물류를 관리하는 소프트웨어 시스템으로, 육상/항공/해상 및 복합 운송 방식의 최적화를 수행합니다.

#### 주요 기능
1. **배차계획 수립**: 최적 배차 계획 자동 생성
2. **실시간 차량정보 확인**: GPS 기반 위치 추적
3. **오더 관리**: 주문 입력 및 상태 관리
4. **운송 계획 및 모니터링**: 배송 진행 상황 실시간 모니터링
5. **비용 관리 및 감사**: 운송료 정산 및 비용 분석
6. **데이터 분석 및 보고**: 성과 분석 및 최적화

#### 연동 가능 시스템
- ERP (전사적 자원관리)
- WMS (창고관리시스템)
- OMS (주문관리시스템)
- FMS (포워더관리시스템)

#### 국내 주요 TMS 솔루션
- **한국네트웍스**: air-TMS (배차계획, 실시간 차량정보)
- **Web2gether**: 컨테이너/벌크 운송 특화 웹 기반 시스템
- **DB Schenker**: 통합 운송관리 서비스
- **C.H. Robinson**: 글로벌 TMS 솔루션

## 5. 입찰 및 배차 방식

### 5.1 전통적 배차 방식

#### 1) 직접 배차 (Direct Assignment)
- **주선업자**가 자사 운송사 또는 협력사에 직접 배차
- **의사결정**: 중앙 배차 담당자가 수동으로 판단
- **장점**: 친분 기반 신뢰성, 운송사 특성 반영
- **단점**: 최적성 보장 부��, 시간 소모

#### 2) 화물정보망 공고 (Public Freight Board)
- **형태**: 오픈 입찰 형태
- **참여자**: 독립 차주, 소규모 운송업자
- **수락 방식**: 차주가 자발적으로 수락
- **수수료**: 주선업자가 화물정보망에 공고료 지불

### 5.2 현대적 자동 배차 방식

#### 1) AI/알고리즘 기반 최적 배차
**구성 모듈**:
1. **후보지 설정 모듈**: 배차 가능한 차량 후보 선정
2. **차량 배치 모듈**: 배차 대기시간 최단화
3. **실시간 교통 분석 모듈**: 최단 시간 도착 차량 산출

**활용 데이터**:
- 실시간 교통 정보
- 기상 데이터
- 차량 위치 정보
- 운전자 운행 특성

#### 2) 빅데이터 기반 운임 산출
예: 티맵 화물의 "최적 운임 조회" 서비스
- **데이터셋**: 110만 개 이상의 운송 빅데이터
- **분석 요소**: 기상, 수요·공급, 도로 상황
- **결과**: 실시간 최적 운임료 자동 산출

## 6. 화물차주(지입차) 오더 수락 방식

### 6.1 지입제 (Jiip System) 개요

#### 정의
영업용 화물차주가 차량과 넘버를 보유하고, 주선업자나 운송사로부터 정기적인 물량을 받아 운송하는 계약 형태입니다.

#### 지입 구조
```
주선업자/운송사
  ↓ (물량 할당)
지입업자 (Lease Operator)
  ↓ (지입료 지불)
개인 차주
```

#### 지입 계약 방식
1. **완제 방식**: 월 고정급여 (일급제, 월급제)
2. **매출제 방식**: 본인이 올리는 매출에 따라 수익을 가져가는 구조

### 6.2 오더 수락 프로세스

#### 온라인 방식 (현대적)
1. **모바일 앱/웹 플랫폼**에서 화물 정보 확인
2. **실시간 오더 조회**:
   - 품목, 출발지/도착지
   - 배송 비용
   - 예상 소요 시간
3. **자발적 수락**: 차주가 원하는 물건을 선택
4. **시스템 확정**: 배차 완료 및 배송 시작

#### 오프라인 방식 (전통적)
1. **전화 또는 무선기**: 주선업자로부터 직접 연락
2. **구두 계약**: 물량, 배송료 협상
3. **서면 확인**: 기록 및 정산

### 6.3 관련 온라인 플랫폼

- **짐차넷**: 전국 화물차 일자리 정보 제공
- **물류원**: 실시간 전국 지입/지입차량/지입일자리 정보
- **전국지입차정보**: 지입차량 정보 제공

### 6.4 지입제의 문제점

#### 주요 이슈
1. **명의 문제**: 개인이 구매한 차량도 소유권이 회사 명의로 등록
2. **지입 사기**: 부실 정산, 불공정한 수수료 공제
3. **법적 보호 미흡**: 개인 차주의 권리 침해
4. **투명성 부족**: 계약서 작성 표준 부재

## 7. 규제 환경

### 7.1 화물자동차운수사업법 (Freight Transport Business Act)

#### 법적 지위
- **목적**: 화물 운송 질서 확립, 운송사업자의 의무 규정
- **주요 규정 영역**:
  - 운송사업 종류 및 요건
  - 차고지 이용 및 운송시설
  - 수송의 안전성
  - 화주의 편의
  - 공정한 경쟁

#### 주선업체 규제
- **새로운 규제**: 플랫폼 운송사업자 관련 조항 추가
- **주요 의무**:
  1. **정보 보고 의무**: 운영 현황 정기 보고
  2. **공정성 확보 의무**: 배차 과정의 투명성 보장
  3. **안전 관리**: 운송 과정의 안전 관리
  4. **분쟁 해결**: 이용자 분쟁 처리 제도

#### 플랫폼 운영자 준수 사항
- 자체 플랫폼 운영 또는 플랫폼 통한 위탁 시 준수 필수
- 공정거래위원회 규정 준수
- 이용약관 공시 의무

### 7.2 관련 법제 동향

#### 최근 개정 방향
- **디지털 플랫폼 사업자 규제 강화**
- **운송사업 투명성 제고**
- **데이터 보호 및 개인정보 관리**

| **주요 고객** | 기업 화주 (700여 개) | 광범위 (화주) | 온라인 쇼핑몰 |
| **배차 방식** | 자사+협력+화물정보망 | 자동 배차 (AI) | 자동화 시스템 |

| **기술 특징** | 데이터 축적 및 분석 | 빅데이터 기반 운임 | RFID 자동화 |
| **지역 커버리지** | 전국 | 전국 8개 지사 | 전국 통합 네트워크 |

## 9. 핵심 인사이트

### 9.1 시장 구조의 변화

#### 과거 (전통)
- 주선업자 중심의 인맥 기반 배차
- 화물정보망을 통한 오픈 입찰
- 낮은 투명성, 높은 운임 변동성

#### 현재 (디지털 전환)
- 플랫폼 기반의 자동 배차
- 빅데이터와 AI를 통한 최적화
- 실시간 추적 및 투명한 정산
- API를 통한 통합 연동

### 9.2 주요 경쟁 요소

1. **배차 성공률**: 낮을수록 좋음 (빠른 배정)
2. **운임 투명성**: 빅데이터 기반 합리적 운임
3. **기술 통합**: API 및 자동화 수준
4. **네트워크 규모**: 차주/운송사 확보
5. **사용자 경험**: 모바일 앱, 웹 접근성

### 9.3 규제 영향

- **플랫폼 사업자의 책임 강화**: 공정성 확보 의무
- **데이터 보호**: 개인정보 관리 강화
- **투명성 요구**: 배차 과정 공개
- **분쟁 해결**: 이용자 보호 강화

## 11. 결론

한국 화물운송 시장은 전통적인 주선업 중심의 구조에서 디지털 플랫폼 기반의 시스템으로 빠르게 전환되고 있습니다.

**핵심 변화**:
1. **자동화**: 수작업 기반 배차 → AI/알고리즘 기반 자동 배차
2. **투명성**: 불투명한 운임 → 빅데이터 기반 합리적 운임
3. **통합**: 독립된 시스템 → API 기반 통합 생태계
4. **규제**: 자유로운 운영 → 공정성 확보 의무

**주요 플레이어**:
- **로지스팟**: B2B 기업 고객 중심, 오픈 API 제공
- **티맵 화물**: 빅데이터 기반 운임 산출, 높은 배차 성공률
- **CJ 대한통운**: 이커머스 물류 통합, 자동화 시스템

이러한 플랫폼들과의 연동을 통해 기업은 더욱 효율적이고 투명한 물류 관리가 가능해지고 있으며, 규제 강화는 시장의 건전성을 높이는 역할을 하고 있습니다.
