- [ ] **지표 사전 정의**
  - [ ] 탄소집약도: gCO₂e/t·km (기준기간, 조건 표준화 규칙)
  - [ ] 활동데이터: 톤·km, 운행거리, 적재율, 공차율, 대기시간 등

- [ ] **데이터 모델(초안)**
  - [ ] Shipper: 사업부/거점/노선/물량/공급사/Fleet/오더
  - [ ] Carrier: 고객(화주)/계약/노선/편대(Fleet)/차량/드라이버/오더
  - [ ] Owner: 차량/운행세션/오더/정산/평판/탄소포인트
- [ ] **UI 컴포넌트 라이브러리 구성(Tailwind)**
  - [ ] KPI 카드, 트렌드 차트, 비교 테이블, 배지(ISO-14083/실측), 필터 바(기간/노선/거점)
  - [ ] 상태 Pill(양호/주의/위험), Empty/Loading/Error states
- [ ] **모킹 데이터 세트 작성**(Figma/JSON 수준) + 시드(기간 14/30/90일)
- [ ] **디자인 원칙**
  - [ ] 비교 가능성 강조: 단위/조건/기간을 항상 화면에 노출
  - [ ] 신뢰성 강조: 산식/근거/커버리지/검증 상태를 1클릭 접근 가능

## 1) 화주기업 대시보드(Shipper)
### 1-1 핵심 질문
- [ ] 우리 **Scope 3(운송)**가 어디서 많이 발생하는가? (거점/노선/공급사/Fleet)
- [ ] 동일 조건에서 **가장 탄소효율이 좋은 운송 옵션**은 무엇인가?
- [ ] 탄소 기준(Threshold/가중치)을 적용했을 때 **비용/리드타임 영향**은?

### 1-2 화면/위젯
- [ ] Overview
  - [ ] Scope 3 추정/집계(기간) + YoY/WoW 변화
  - [ ] 평균 탄소집약도(gCO₂e/t·km) + 목표 대비
  - [ ] 상위 배출 기여 Top 5(노선/거점/공급사)
- [ ] Fleet 비교/구매 의사결정
  - [ ] Fleet 리스트: 탄소집약도/가격/OTIF/데이터 신뢰도(ISO/실측/커버리지)
  - [ ] 조건 필터: 노선, 톤수, 차량타입, 시간대, 거리 구간
  - [ ] 정책 시뮬레이터: Threshold 적용 시 선택 가능한 Fleet 수/비용 영향
- [ ] 리포팅/감사
  - [ ] 근거 데이터/산식 버전/감사 로그 다운로드(목업)

### 1-3 액션(CTA)
- [ ] “탄소 기준 입찰 생성” / “그린라벨 오더 발행” 버튼
- [ ] 기준 미달 선택 시: 승인 워크플로(예외 승인) UX

## 2) 물류기업/주선사 대시보드(Carrier/Broker)
### 2-1 핵심 질문
- [ ] 우리 편대(Fleet)가 화주에게 **프리미엄(저탄소) 가치**를 얼마나 제공하나?
- [ ] 어떤 노선/차량군에서 탄소집약도가 악화되는가?
- [ ] ISO-14083/실측 데이터를 어떻게 **신뢰 자산**으로 보여줄까?

### 2-2 화면/위젯
- [ ] Sales/Account
  - [ ] 화주별: 탄소집약도/OTIF/클레임/매출(또는 물량) 요약
  - [ ] 그린라벨 오더 수주 현황(파이프라인)
- [ ] Fleet 성과
  - [ ] Fleet별 탄소집약도 분포 + 개선 추이
  - [ ] 데이터 커버리지(%) / 검증 배지 / 결측 알림
- [ ] 운영 개선 추천
  - [ ] 공차/대기/적재율 개선 기회 Top
  - [ ] 권장 액션(차량 배정/경로/운행 교육) 카드

### 2-3 액션(CTA)
- [ ] “탄소 성과 리포트 공유(화주용)” 버튼
- [ ] “검증(ISO/실측) 상태 업데이트” 흐름(목업)

- [ ] 오더/수익
  - [ ] 오더 리스트(그린라벨 배지) + 예상 수익/정산 상태

- [ ] 리워드
